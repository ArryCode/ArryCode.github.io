三、TensorFlow入门

3.1 TensorFlow 的计算模型——计算图

TensorFlow 是通过一个计算图的形式来表述计算的编程系统。TensorFlow 中每个计算都是计算图上的一个节点，而节点之间的边描述了依赖关系。

  • 计算图的使用

除了默认的计算图，TensorFlow 支持通过 tf.Graph 函数来生成新的计算图，不同图上的张量和运算不会共享。

TensorFlow 中的计算图不仅仅可以用来隔离张量和计算，它还可以提供管理张量和计算的机制。计算图可以用 tf.Graph.device 函数来指定设备。

在一个计算图中，通过集合（collection）来管理不同类型的资源。

3.2 TensorFlow 的数据模型——张量

从功能的角度，张量（tensor）可以简单认为是向量。

一个张量中主要保存了三个属性：名字（name）、维度（shape）和类型（dtype）。

  • 张量的命名：node:src_output，node为节点的名称，src_output 表示当前张量来自节点的第几个输出。
  • 张量的维度：数字是几就代表是几维向量
  • 张量的类型：类型唯一，且参与运算的张量类型要一致，不然程序会出错。TensorFlow支持14种不同类型，主要包括了实数（ tf.float32，tf.float64），整数（tf.int8 16 32 64，tf.uint8），布尔型（tf.bool）和复数（tf.complex64，tf.complex128）

张量的作用：

  • 对中间值结果的引用，提高代码可读性
  • 当计算图构造完成后，可以用来计算结果

3.3 TensorFlow 的运行模型——会话

TensorFlow用会话来执行定义好的运算，会话拥有并管理TensorFlow程序运行时的所有资源。在所有计算完成之后需要关闭会话帮助系统来释放资源，否则会出现资源泄露的问题。

TensorFlow中使用会话的方式一般两种：

  • 明确调用会话生成函数和关闭会话函数

    使用这种模式时，在所有计算完成后必须关闭明确调用 Session.close 函数来关闭会话并释放资源。但是如果程序遭遇异常，就无法关闭，所以会有第二种模式。

  • 通过Python上下文管理器来使用会话

TensorFlow也可以指定默认的会话，当默认的会话被指定后，可以通过 tf.Tensor.eval函数来计算一个张量的取值。还提供在交互式环境下直接构建默认会话的函数 —— tf.InteractiveSession。

用ConfigProto Protocol Buffer 来配置需要生成的会话，tf.ConfigProto()

3.2 None值

在 Python 中，None表示没有值，None是 NoneType 类型中的唯一值。当函数没有设置返回值时，Python会在末尾加上 return None 。

3.3 关键字参数和print()

可以设置print()函数的关键字参数，来制定打印格式。

  • end 指定末尾打印什么，不设置的情况下，print() 函数会自动在传入的子串末尾添加了换行符。

  • sep 自定义分割符，不设置的情况下，print() 函数会自动在传入的子串末尾添加了换行符。

3.4 局部和全局作用域

局部变量：在被调用函数内赋值的变量和变元，处在该函数的 ”局部作用域“ ，被称为局部变量。

全局变量：在所有的函数之外赋值的变量，属于 “全局作用域”，被称为全局变量。

  • 局部变量不能在全局作用域内使用
  • 局部作用域不能使用其他局部作用域内的变量
  • 全局变量可以在局部作用域中读取
  • 局部变量和全局变量的名字可以相同
