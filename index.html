Machine Learning for Android Malware Detection Using Permission and API Calls（2013）

Posted on 2019-07-19 |

Machine Learning for Android Malware Detection Using Permission and API Calls（2013）

理论依据

Android权限设置

恶意APP倾向于请求比良性APP更多的权限。

为了便于机器学习，作者用二进制的形式来表示APP权限的请求。设二进制向量 $P$ 表示一个APP申请权限，如果$P_i = 0$，则APP没有请求第 $i$ 申请，反之则申请。

Android API调用

经过作者调研，良性APP使用API调用比恶意APP要多。

同样用二进制表示，设二进制向量 $A$ 代表一个APP的API调用，若 $A_i = 0$，则该APP没有调用第 $i​$ 个API，反之则调用。

研究方法

graph LR
恶意APP与非恶意APP混合 --> APP分析器
APP分析器 --> 描述特性
描述特性 --> 提取特征
提取特征 --> 机器学习算法分类
机器学习算法分类 --> 恶意APP与非恶意APP分开

实验用了不同的机器学习算法，如SVM、决策树、集成学习算法Bagging，比较后，最后得出Bagging最优。

总结

文章提出了一种检测恶意软件的方法，但是单单从权限请求多少和API调用多少来看，可能不是特别全面。另外，APP比较大的话，生成的二进制文件的工作量就比较大。文章也没有提出用机器来对特征进行提取。

Android APP结构

Android APP的所有必要文件都被打包进一个APK压缩文件中。APK文件通常含有下面几项：

  • res文件夹。存放可编译的资源文件，如布局文件等。
  • asset文件夹。存放一般资源文件，如图片等。
  • classes.dex文件。包含应用程序的字节码（bytecode），是可执行代码的主要来源，尽管有一些方法可以加载外部代码或从APK的本地库包（native library package）中加载代码。
  • AndroidManifest.xml文件。
  • lib文件夹。存放本地库文件。
  • 其它文件
